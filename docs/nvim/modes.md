# 基本概念: モード

vimの最大の特徴は**モード**の存在です。VSCodeにはない概念で、最初はここで混乱します。

## モードの種類

```
                    i / o / a
                  ──────────→
  ノーマルモード               挿入モード
  (移動・操作)   ←──────────  (文字入力)
                     Esc

                    v / V
                  ──────────→
  ノーマルモード               ビジュアルモード
  (移動・操作)   ←──────────  (テキスト選択)
                     Esc

                      :
                  ──────────→
  ノーマルモード               コマンドモード
                 ←──────────  (コマンド入力)
                  Enter / Esc
```

## ノーマルモード

**neovimを開いたとき、最初はノーマルモードです。**

- 文字を入力しても画面に表示されない（コマンドとして解釈される）
- カーソル移動や編集操作の起点となるモード
- 迷ったら `Esc` を押すと必ずここに戻れる

## 挿入モード (Insert Mode)

文字を実際に入力するモード。VSCodeの通常の編集状態に相当します。

画面左下に `-- INSERT --` と表示されているときが挿入モードです。

### ノーマルモード → 挿入モード

| キー | カーソル位置 |
|------|------------|
| `i` | カーソルの左から入力 |
| `a` | カーソルの右から入力 |
| `I` | 行頭から入力 |
| `A` | 行末から入力 |
| `o` | 下に新しい行を作って入力 |
| `O` | 上に新しい行を作って入力 |
| `Ctrl+Enter` | 下に新しい行を作って入力 (VSCode風) |
| `Ctrl+Shift+Enter` | 上に新しい行を作って入力 (VSCode風) |

### 挿入モード → ノーマルモード

| キー | 動作 |
|------|------|
| `Esc` | ノーマルモードに戻る |
| `Ctrl+[` | ノーマルモードに戻る（Escの代替） |

## ビジュアルモード (Visual Mode)

テキストを選択するモード。VSCodeのクリック&ドラッグに相当します。

| キー | 選択の種類 |
|------|----------|
| `v` | 文字単位で選択 |
| `V` | 行単位で選択 |
| `Ctrl+v` | 矩形選択（列選択） |

選択後は `y`（コピー）、`d`（切り取り）、`>`（インデント）などの操作が使えます。

## コマンドモード (Command Mode)

ノーマルモードで `:` を押すと画面下部に入力欄が現れます。

```
:w          保存
:q          終了
:wq         保存して終了
:q!         保存せず強制終了
```

詳しくは[ファイル・バッファ操作](./files.md)を参照してください。

## よくある混乱とその対処

| 症状 | 原因 | 対処 |
|------|------|------|
| 文字を打っても画面に出ない | ノーマルモードにいる | `i` を押して挿入モードへ |
| 打った文字が変な動作をする | ノーマルモードでvimコマンドが実行されている | `Esc` → `u` でアンドゥ → `i` で入力 |
| `:` 以降に文字が出てくる | コマンドモードにいる | `Esc` でキャンセル |
| 何も動かなくなった | マクロや待機状態の可能性 | `Esc` を数回押す、または `Ctrl+c` |
